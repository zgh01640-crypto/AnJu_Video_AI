## 一、产品概述

### 1.1 背景与目标

安居集团在工程建设管理中，需要对施工现场拍摄的视频资料进行及时分析，掌握施工专业类别、形象进度及现场状态。传统人工查阅耗时费力，本系统借助多模态大模型（Qwen3-VL-Plus）对工程视频进行智能分析，快速输出施工内容判断与进度评估，提升工程管理效率。

### 1.2 产品定位

面向安居集团工程管理人员的**内部工具系统**，实现工程视频的上传、存储、播放与 AI 智能分析一体化，辅助工程进度管控与资料留存。

### 1.3 核心价值

- **效率提升**：替代人工逐帧查阅，秒级获取施工分析报告
- **标准化输出**：统一的分析维度，支持管理层快速决策
- **资料沉淀**：视频与分析结果持久化存储，支持历史追溯

### F1 视频上传模块

#### 3.1.1 功能描述

用户通过拖拽或点击选择的方式，将工程视频上传至阿里云 OSS，系统在上传过程中显示进度反馈。

#### 3.1.2 详细需求

**支持格式**
- MP4（推荐，兼容性最佳）
- MOV（Apple 设备常见格式）
- AVI（传统录像设备格式）

**文件限制**
- 单文件大小上限：200MB
- 超出限制时，系统在选择阶段（前端校验）即阻止上传并提示错误

**交互流程**
1. 用户进入系统，默认展示上传区域（拖拽 Drop Zone）
2. 用户拖入或点击选择文件
3. 前端校验文件格式与大小
   - 校验不通过 → 弹出错误提示，禁止继续
   - 校验通过 → 进入上传流程
4. 系统调用 OSS 预签名上传接口，开始上传
5. 上传过程中展示：进度条 + 已上传大小/总大小 + 百分比
6. 上传完成 → 自动跳转至视频播放 & 分析界面

**错误处理**
| 错误场景 | 提示文案 |
|----------|----------|
| 格式不支持 | "仅支持 MP4、MOV、AVI 格式，请重新选择" |
| 文件超过 200MB | "文件大小超出限制（最大 200MB），请压缩后重新上传" |
| 网络中断 | "上传失败，请检查网络连接后重试" |
| OSS 服务异常 | "存储服务暂时不可用，请稍后重试" |

### F2 视频播放模块

#### 3.2.1 功能描述

视频上传完成后，系统通过 OSS 签名 URL 提供在线播放能力，使用浏览器原生播放器或 HTML5 Video 组件实现。

#### 3.2.2 详细需求

**播放功能**
- 播放 / 暂停
- 进度条拖拽跳转
- 音量调节
- 全屏模式
- 播放速度调节（0.5×、1×、1.5×、2×）

**展示规格**
- 播放器区域固定高宽比 16:9
- 视频加载中显示骨架屏 Loading 动画
- 视频封面帧取第一帧自动截图作为预览图

**URL 安全**
- 播放 URL 通过 OSS 签名生成，有效期 2 小时
- 过期后用户刷新页面自动续签

### F3 分析提示词管理模块

#### 3.3.1 功能描述

系统提供可编辑的分析提示词输入区，用户可在默认提示词基础上修改，以获得针对性的分析结果。

#### 3.3.2 详细需求

**默认提示词**
```
请分析工程视频，正在进行哪些专业施工，形象进度如何
```

**编辑能力**
- 文本域可自由编辑，支持换行
- 字数上限：500 字
- 实时显示剩余字数
- 提供"恢复默认"按钮，一键还原默认提示词

**提示词建议（UI 快捷标签）**  
系统提供以下常用分析方向的快捷标签，点击追加到提示词末尾：

| 标签 | 追加内容 |
|------|----------|
| 安全隐患 | ，并指出画面中是否存在安全隐患 |
| 人员数量 | ，统计视频中出现的施工人员数量 |
| 材料使用 | ，描述使用的主要建筑材料 |
| 机械设备 | ，列举出现的施工机械与设备 |

### F4 AI 视频分析模块

#### 3.4.1 功能描述

用户点击"开始视频分析"按钮，系统将视频（OSS URL）与提示词提交至 Qwen3-VL-Plus 模型，获取分析结果并实时流式返回展示。

#### 3.4.2 详细需求

**触发条件**
- 视频已上传成功（OSS URL 有效）
- 提示词不为空
- 当前无正在进行的分析任务

**分析按钮状态**

| 状态 | 按钮文案 | 按钮样式 |
|------|----------|----------|
| 可用 | 开始视频分析 | 主色填充，可点击 |
| 分析中 | 分析中… | 加载动画，禁用 |
| 重新分析 | 重新分析 | 次级样式，可点击 |

**API 调用参数（Qwen3-VL-Plus）**

```json
{
  "model": "qwen-vl-plus",
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "video",
          "video": "<OSS_SIGNED_URL>"
        },
        {
          "type": "text",
          "text": "<用户提示词>"
        }
      ]
    }
  ],
  "stream": true
}
```

**流式输出**
- 支持 Server-Sent Events（SSE）流式返回
- 分析结果逐字/逐段实时渲染到结果区域
- 展示"正在分析..."动态光标动画

**超时与重试**
- 单次分析超时时间：120 秒
- 超时后提示"分析超时，请重试"，按钮恢复可用状态
- 不自动重试，由用户手动触发

---

### F5 分析结果展示模块

#### 3.5.1 功能描述

AI 返回的分析内容以结构化的卡片形式展示，支持 Markdown 渲染，并提供复制与导出功能。

#### 3.5.2 详细需求

**展示内容**
- 分析时间戳（精确到秒）
- 分析所用提示词（折叠展示）
- 分析正文（Markdown 渲染，支持标题、列表、加粗）
- 分析耗时

**操作功能**
- 一键复制全文
- 导出为 .txt 文件
- 清空结果（需二次确认）

**历史记录（页面级，不持久化）**
- 同一会话内可保留最近 3 次分析结果
- 以 Tab 形式切换查看

## 四、非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 视频上传速度 | 取决于用户带宽，进度实时反馈 |
| 视频首帧加载 | ≤ 3 秒（正常网络环境） |
| 分析首字符返回 | ≤ 10 秒（模型响应） |
| 分析总耗时 | 预期 30~90 秒（视视频时长） |

### 4.2 安全需求

- OSS 访问使用临时签名 URL，不暴露永久密钥
- 前端不存储任何 AccessKey/SecretKey
- API Key 通过后端代理传递，不在前端硬编码
- 上传文件仅限指定 Bucket，禁止越权访问

### 4.3 兼容性需求

| 浏览器 | 版本要求 |
|--------|----------|
| Chrome | ≥ 90 |
| Edge | ≥ 90 |
| Firefox | ≥ 88 |
| Safari | ≥ 14 |

- 视频格式需兼容浏览器原生解码能力，MP4/H.264 为主要推荐格式

### 4.4 可用性需求

- 系统可用性：99%（工作日工作时间）
- OSS 存储按阿里云 SLA 保障

## 八、开发实现说明

### 8.1 实现方案

本系统使用 **Claude Artifacts（claude-opus-4-6）** 构建，通过 Anthropic API 驱动前端界面与 AI 能力编排。

**前端**：React + Tailwind CSS，运行在 Claude Artifact 沙箱中

**关键实现要点**：

1. **OSS 直传**：前端通过后端换取临时凭证（STS Token）或预签名 URL 直传 OSS，避免流量经过应用服务器
2. **视频分析**：通过 Anthropic API 将 OSS 视频 URL 和提示词发送给 Qwen3-VL-Plus
3. **流式渲染**：使用 SSE 流式接收模型输出，实时 append 到结果区域
4. **视频播放**：使用 HTML5 `<video>` 标签加载 OSS 签名 URL

### 8.2 环境配置项

| 配置项 | 说明 |
|--------|------|
| `OSS_REGION` | OSS 地域，如 `oss-cn-hangzhou` |
| `OSS_BUCKET` | Bucket 名称 |
| `OSS_ACCESS_KEY_ID` | 阿里云 AccessKey ID（建议使用 RAM 子账号） |
| `OSS_ACCESS_KEY_SECRET` | 阿里云 AccessKey Secret |
| `QWEN_API_KEY` | 通义千问 API Key |
| `ANTHROPIC_API_KEY` | Anthropic API Key（claude-opus-4-6） |

## 九、验收标准

### 9.1 核心功能验收

| 验收项 | 通过标准 |
|--------|----------|
| 视频上传 | MP4/MOV/AVI 格式，≤200MB 可成功上传并获得 OSS URL |
| 格式拦截 | 非支持格式或超大文件在前端被拦截，显示正确错误提示 |
| 视频播放 | 上传完成后视频自动可播放，支持基础播放控制 |
| 默认提示词 | 系统初始展示指定默认提示词 |
| 提示词编辑 | 用户可修改提示词，恢复默认功能正常 |
| 分析触发 | 点击按钮后进入分析状态，按钮禁用，显示加载动画 |
| 流式返回 | 分析结果逐步显示，非一次性全量出现 |
| 分析完成 | 结果完整展示，按钮恢复为"重新分析" |
| 复制导出 | 复制到剪贴板和导出 txt 功能正常 |

### 9.2 异常场景验收

| 场景 | 预期结果 |
|------|----------|
| 上传 200MB+ 文件 | 前端拦截，显示大小超限提示 |
| 上传 .mp4 以外格式（如 .wmv） | 前端拦截，显示格式不支持提示 |
| 分析超时（模拟） | 显示超时提示，按钮恢复可用 |
| 提示词清空后点击分析 | 按钮不可点击或提示"请输入分析提示词" |
